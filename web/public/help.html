Personal Brain OS · Field Manual · v3

How to live with it —The Operating Manual

The system's job: you speak, it remembers, it files, it resurfaces. Your job is four habits and two rituals. Sections 01–06 are today; 07–09 are what each coming pass adds to your day, so you always know what's next and why.

01 · The Daily Loop

What you do vs. what it does

Teal ticks = your moves. Grey ticks = automatic. Total human time: under 10 minutes a day.

You · all day, anywhere
Capture

Thought arrives → press Action Button → talk → press again. That's the entire habit. Say the tag in the first five words if you know it ("Idea: …", "Todo: …", "Resource: …") and routing costs nothing. Don't organize. Don't type. Don't open Obsidian.

No Action Button? Lock-Screen Voice Memo control, Control Center wave icon, or "Hey Siri, record a voice memo" — all record with the screen locked.

You · 2 taps per memo, batch anytime
Send to Brain

Voice Memos → share → → Brain → optional name, optional tag → "✅ Captured". Recordings are safe in Voice Memos until you do this; batching at lunch or evening is fine. Capture never waits on filing.

System · Mac, every 5 min
Pipeline

Syncthing carries the file over → whisper.cpp transcribes locally → tags route free, untagged goes to Haiku → schema-correct note lands in the right folder → action items appended to today's todos → audio archived, never deleted. Low-confidence guesses park in 00-Inbox instead of being silently misfiled.

You · morning, 2 minutes
Glance + Triage

Open the Cockpit on your home screen (Pass 3 — it shipped; _System/PIPELINE-STATUS.md remains the markdown fallback). The Today screen answers the three questions at a glance: the traffic-light card is calm, teal, or pink; the streak shows yesterday landed. Then the Triage tab — approve or retype, one tap per note.

System · when something breaks
It tells you

Failures go to _failed/ with a plain-English reason and one ntfy push. Silence means healthy. You never re-listen to audio "just to check".

You · Sunday, 15 minutes
Weekly sweep

Inbox to zero · skim capture_log.md · git log the vault · one voice reflection on the week. Done.

02 · The Don'ts

What you never do

Each of these feels productive. Each one is the system eating your life instead of serving it.

  • Don't hand-file or reorganize notes in Obsidian. The pipeline files; you only triage 00-Inbox.
  • Don't edit anything in raw/. Transcripts are immutable sources. Synthesis happens in wiki/, in the weekly compile.
  • Don't invent new folders, types, or tags mid-flow. Vocabulary is locked. Taxonomy itches go to IDEAS-BACKLOG.md.
  • Don't skip a capture because it "isn't important enough." Capture everything; classification is the machine's call.
  • Don't build past the current pass. New feature ideas → DEFERRED.md, one line, keep living.
  • Don't let 00-Inbox pass ~10 items. Past ten, triage becomes a chore; chores die.
  • Don't babysit the pipeline. Fresh heartbeat + quiet ntfy = working. Checking is the old anxiety, not the new system.
03 · First Run

Get it live — per tool

One-time, top-to-bottom, one sitting (~90 min). The code is built; this is the wiring.

Mac

whisper.cpp

  • brew install whisper-cpp (or clone + make)
  • Download small.en model (~466 MB)
  • Test on one wav; note binary + model paths
  • brew install ffmpeg if missing
Repo

config.json

  • Copy config.example.json → config.json
  • Fill vault, inbox (Syncthing CaptureInbox), archive, failed paths
  • whisper binary + model paths
  • Random string as api.auth_token; threshold stays 0.7
Shell

API keys

  • export ANTHROPIC_API_KEY=… in ~/.zshrc
  • OPENAI_API_KEY only for the cloud fallback
  • Keys live in env — never in config, never in git
Vault

Git

  • git init the vault if not already
  • One clean commit before the backlog run
  • This is the undo button for every batch and every purge
Phone + Mac

ntfy

  • Install ntfy on iPhone, subscribe to a unique topic
  • Put url + topic in config.json
  • Send a test push; feel the buzz
Sync

Syncthing + Möbius

  • Confirm CaptureInbox syncs phone ↔ Mac
  • Drop a test file on the phone; watch it land on the Mac
  • Möbius folder visible in the Files app
iPhone

Capture triggers

  • Action Button → Voice Memo (records while locked)
  • Build "→ Brain": accept audio → optional name → tag menu (8 tags, skippable) → rename YYYY-MM-DD-HHmm name #tag → save to CaptureInbox → ✅
  • Test end-to-end: record → share → file on Mac
Terminal

Run it

  • First: the Backlog Ritual (04)
  • Daily: .venv/bin/python -m pipeline --loop in a tab that stays open
  • Phase 5 makes this permanent on ZimaOS
Browser

Cockpit (Pass 3)

  • cd web && npm ci && npm run build, serve dist/ (or npm run dev)
  • Connect screen: server address + the api.auth_token from config.json
  • Install to home screen (browser menu → Add to Home Screen)
  • Settings (gear, top right): connection, deep links, where every key lives
Mac

API (Pass 2 — next build)

  • .venv/bin/uvicorn api.main:app — the server the Cockpit talks to
  • Until it's built, the Cockpit shows its designed "can't reach the server" card — expected, not broken
  • The contract it must implement: web/API-CONTRACT.md
04 · The Backlog Ritual

Your month of audio, finally heard

--backlog
cd brain-cockpit
.venv/bin/python -m pipeline --backlog

Oldest-first, batches of 25. Before each batch it git-commits the vault; after each it prints file → type → destination → confidence and waits. Skim, spot-open two or three notes in Obsidian, press enter. Low-confidence items park in 00-Inbox — triage across the week, not tonight.

Expect 2–4 sittings for a month. Failures land in _failed/ with a reason — retry after fixing, never mid-run.

The day the backlog clears, the 30-day clock starts. Not before.

06 · The Gate

The 30-day test

From the day the backlog clears, live the loop daily. The system passes only if, by day 30, you can name out loud:

3 decisions it improved 3 insights it saved

Fail → kill down to the parts you actually used. Not opened daily by week 3 → same. A tool, not a monument.

07 · The Cockpit

Pass 3 is live; what 4–5 add next

Same loop, better instrument. Nothing below changes the habits — it removes friction from them.

Pass 3 · Cockpit UI · shipped
Glance is an app

The morning check is a PWA on your home screen: traffic-light health card (calm / teal "needs your call" / pink "needs a look"), capture streak in big numerals with the 30-day dot row, one resurfaced note with an Open-in-Obsidian link, the triage queue as one-tap cards, quick text capture with the 8 tag chips, and a Settings screen (gear icon) with the connection, deep links, and a map of where every key and path lives. You do: install it to the home screen once; the 2-minute morning ritual now takes 60 seconds. It needs the Pass 2 API running to have data — until then it shows the plain-English "can't reach the server" card.

Pass 4 · Integrations
One pane of truth

Status cards for every moving part — whisper.cpp, OpenAI fallback toggle, Claude key, ntfy, Syncthing, git, watcher heartbeat — plus deep links out to Obsidian, Dex, Gmail, Calendar, cal.diy, n8n, ZimaOS. You do: nothing daily; visit when something feels off instead of guessing.

Pass 5 · Hardening
It watches itself

Watchdog pings if the watcher dies, optional 8am digest, auto-retry on transient failures, startup self-check, backup card. You do: trust the silence even more.

08 · Resource OS · Pass 6

Your library, alive

Every tool, book, film, article, recipe, and place you save — as a visual gallery with your one-line insight attached. The pink dot means a resource carries an insight.

You · when you meet a resource
Save it by voice

Action Button → "Resource: the book Alchemy by Rory Sutherland — someone said it's the anti-logic of branding." Done. The pipeline files it as a resource note; the gallery picks it up on next load. Typing a URL into quick capture works too.

You · browsing, deciding, remembering
The gallery

Cover-card grid, filter by category (books / films / tools / recipes / places) and status, search everything. Tap a card → detail drawer: Open source ↗ deep-links out, Open in Obsidian for the raw note, one button advances the status ("Start consuming" → "Mark consumed").

You · the moment you finish something
One takeaway

Marking a resource consumed lands you in a single input: "One takeaway?" Speak-to-type one sentence, save. That sentence is the whole point of the Resource OS — a library of things you consumed with what they changed in you attached. Skippable, never nagging.

System · sample data lifecycle
Seeded → explored → purged

The dashboard launches with ~36 sample resources (sample: true) so you learn it full, not empty. When real resources flow in: the quiet Sample data card at the bottom removes them — scoped to older than 1 day / 1 week / 1 month / all, git-committed before every purge, physically unable to touch a real note. The card disappears when no samples remain.

You · Sunday, folded into the sweep
Shelf check

Filter status = consuming. Anything stalled three weeks? Be honest: archive it or finish it. The library stays alive because it stays honest.
